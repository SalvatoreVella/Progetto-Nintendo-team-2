Utilizzare un solo tag h1 all'interno dell'html.

Usare quanto più possibile i tag semantici.

Aggiungere sempre un attributo alt al tag img.

Nel caso si utilizzi il tag picture specificare l'attributo type all'interno del tag source.

Tutti i media del progetto devono essere inseriti all'interno della cartella assets.

Evitare di utilizzare stili CSS inline.

Evitare di utilizzare !important.

Come unità di misura preferire i "rem" e gli "em" ai "px".

Utilizzare nomi per le classi in kebab-case.

Utilizzare nomi per le classi in inglese.

I nomi delle classi devono avere un significato relativo al contenuto o
 al compito che svolgono, non utilizzare nomi privi di significato ad es. Class:"classe1".

Per lavorare al progetto creare branch a partire da develop.

Utilizzare il branch develop solo per effettuare le pull request.

Controllare periodicamente se sono presenti modifiche nel branch develop ed 
eventualmente fare il merge nel branch in cui si sta lavorando.

I branch devono essere nominati con feature/nome-della-feature.
